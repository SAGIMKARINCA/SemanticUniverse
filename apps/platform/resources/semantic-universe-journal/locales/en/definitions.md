# Semantic Universe Definitions Register

This file keeps short and processable summaries of the main definitions received from the user so far.

## Semantic Universe

Definition:
- The main upper structure that carries all sub-products, resources, decision models and software projects.

## DEPE

Definition:
- `Digital Enterprise Precess Execution`.
- The second core philosophy developed by the user.
- Its detailed definition will be expanded later.

## Object

Definition:
- One of the two primary sets of the universe.

## Event

Definition:
- One of the two primary sets of the universe.
- It is explained through processes.

## Resource

Definition:
- Transition from object to resource occurs when the existence information, function information, activity information and target information of an object are defined together.

## Resource classes

- Human.
- Movable.
- Immovable.
- Time.

## Resource layers

- Base.
- Derivative.
- Derived.
- Functional.

## Process

Definition:
- The way in which the event universe is explained.
- A process logic based not only on outcome, but also on energy, action and flow.

## Process classes

- Existence and management process.
- Core work process.
- Support-service process.
- Measurement, evaluation and development process.

## God Mode

Definition:
- The shell mode that shows the super-admin and founder management area.

## System Main Loader

Definition:
- The first technical role of Semantic Universe on the Laravel platform foundation.

## Shared-memory writing standard

Definition:
- All user-facing texts in shared-memory records are written with proper Turkish characters, capitalization and punctuation.
- Bullets and sentences begin with capital letters and end with punctuation appropriate to their meaning.
- All user-facing files are stored as UTF-8; no text containing broken characters is released.
- Turkish equivalents are preferred in user-facing technical expressions whenever not strictly necessary otherwise.

## Related-source recording

Definition:
- Any file, presentation, determinant document, educational explanation, defining text or reference content provided by the user in relation to a task is recorded under Related Sources in the related history detail with file path and/or explanation.
- This recording area is a mandatory checkpoint for all future actions.

## Server source archive

Definition:
- A protected file area used so that documents and files provided by the user throughout the process can be opened securely through the history/journal layer.
- Every source in this archive has a clickable link in its detail record, its original local path and a short explanation.

## Source document catalog

Definition:
- The catalog layer in which presentations, PDFs and similar source files provided by the user throughout the process are stored inside journal/history together with topic, role and strategic usage purpose.
- The sources in this catalog are preserved for use as references for principles, values, strategic methods and roadmaps in later rounds.

## Multilingual interface layer

Definition:
- The visible language layer in which user-facing texts in Semantic Universe are managed through a common key set for Turkish, English and other languages to be added later.
- In this layer, visible texts are not embedded directly into pages; they are called through language files and locale selection.
- Surfaces such as the main shell, history and sources behave together according to the same language selection.

## Multilingual content layer

Definition:
- The layer in which content documents such as history, decisions, definitions, experiments and source metadata are loaded through locale-based files or translation manifests.
- This layer goes beyond interface translation and changes the page content itself according to the selected language.
